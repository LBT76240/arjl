Protocoles

ARJL permet de simuler plusieurs protocoles disponibles dans les réseaux réels, en voici une liste (sujette à changements) :

Protocole IP :

Le protocole IP est de niveau 3 dans le modèle OSI et permet un service d'adressage pour chaque équipement connecté. Ainsi, chaque interface possède sa propre configuration IP.

Protocole ARP :

Le protocole ARP est à l'interface entre le niveau 2 et niveau 3 du modèle OSI. Il permet de gérer le trafic entre les appareils.

Protocole DHCP :

Le protocole DHCP se situe à la couche 7 du modèle OSI, il permet une configuration automatique des paramètres IP des machines.

Protocole RIPv2 :

Le protocole RIPv2 est un protocole qui permet aux routeurs de s'échanger leurs tables de routage. En cas de modification du réseau, (coupure d'un lien, ou ajout d'un appareil) ce protocole permet de ne pas avoir à éditer les tables de routage manuellement. Cependant, il reste relativement lent par rapport au protocole OSPF.

Protocole OSPF :

Le protocole OSPF a le même rôle que RIP. Il est cependant plus performant.

Ce protocole n'est pas encore implémenté dans ARJL.
